# 小红书封面与内容格式指南

来源：2026-04-01 Wendy 发现页 feed 截图分析 + 高互动账号拆解

## 通用规格

- **尺寸**：1080 × 1440px（3:4 竖版），信息流占据最大面积
- **文字占比**：≤ 40% 画面面积
- **0.8 秒法则**：用户滑动速度决定了封面必须在 0.8 秒内传达核心信息

---

## 类型 1：真人出镜 + 大字叠加（互动天花板最高）

### 代表账号
- **早起虫子**（1.6 万赞）：真人 + 白板 + 大字标题
- **千狐商业增长**（1433 赞）：真人半身照 + 黄色大字
- **蝴蝶号讲师**（9905 赞）：真人面部特写 + 红色大字
- **dontbesilent**（871 赞）：虽然有真人但更偏类型 2

### 视觉特征
- 真人半身照或口播视频截帧作为背景
- 2-3 行大字叠加在画面上，占画面 40-60%
- 文字颜色：白色 / 黄色(#FFD700) / 红色(#e94560)，**必须有描边或阴影**保证可读性
- 人脸通常在画面下半部分或侧面，文字在上半部分

### 标题公式
- 「停止 X」：停止混沌地活着、停止乖乖上班、停止疲惫状态
- 「X 的 Y 个方法」：赚钱最快的 5 个行业
- 「数字 + 反直觉结论」：一年读书七十本 是我最穷的两年
- 「疑问句」：抖音跟小红书最大的区别是？

### 为什么互动最高
- **人脸是信息流里最强的停留信号**——人天生会看脸
- 大字让 0.5 秒内知道内容主题
- 人脸 + 观点 = 人格信任，促进关注（CES 权重 8）

### 适用场景
- 有抖音口播视频可以截帧
- 愿意出镜
- 内容是观点/经验分享类

### 技术实现
- 从视频文件中提取一帧作为背景（选表情生动的帧）
- 或使用抖音下载的 cover.jpg 作为背景
- 叠加 2-3 行大字标题
- 文字需要 text-shadow 或描边保证在任何背景上可读

## 类型 2：纯文字大字报（无需出镜，高互动）

### 代表账号
- **dontbesilent**（694-871 赞）：深色背景 + 白色大字
- **Adeline**（1668 赞）：浅黄便签背景 + 黑色大字 + 黄色高亮
- **康村 Quant 李小姐**：皱纸纹理背景 + 蓝色关键词

### 视觉特征
- 纯色或纹理背景（深蓝 #1a1a2e / 皱纸 / 米白 / 便签黄）
- **三行四字法**：每行 4-6 个中文字，共 2-3 行
- 字号 72-96px，粗体
- **关键词用不同颜色高亮**（黄色下划线、蓝色、红色）
- 无人脸、无图片、无复杂设计元素

### 标题公式
- 「有一种钱 越赚越穷」
- 「一万条推文 装进你的AI」
- 「问得越多 赚得越少」
- 「阿里P11 也没那么难钓嘛」

### 为什么有效
- 纯靠文案力量，0.5 秒传达一个反直觉观点
- 制作成本极低，可以完全自动化
- 不依赖出镜，适合批量产出

### 适用场景
- 无出镜素材时的默认选择
- 内容是商业认知/金句/反直觉观点类
- 批量生产封面

### 技术实现
- HTML 模板 + Chrome Headless 截图
- 背景：纯色渐变或纹理图
- 文字居中，三行排列
- 可选：关键词高亮色

## 类型 3：截图/证据型（中高互动，信任驱动）

### 代表案例
- **web3 常胜君**（82 赞）：加密资产截图 639,500 USD
- **闺蜜老公被裁**（13 赞）：字节跳动期权截图 3470 万
- **顺金融 OE**（90 赞）：收款通知 ¥86,620
- **shona 的财富**（1865 赞）：个人存款 1144 万

### 视觉特征
- 手机截图、交易界面、银行余额、收款通知
- **数字是核心视觉元素**——大额金钱 = 强停留
- 真实感 > 设计感
- 通常是手机全屏截图，3:4 自然适配

### 标题公式
- 直接用数字：「个人存款 1144 万」「撬动 6864 万资产」
- 反差感：「这种真够脏的，我是打死不坏了」

### 适用场景
- 有真实数据/成果可以展示
- AI 使用时长、工具效率对比、收入截图
- 你的 Claude 2000 小时使用截图就属于这类

### 技术实现
- 不需要模板——直接用真实截图
- 可以在截图上叠加高亮标注（红框、箭头）

## 类型 4：Notes 截图/手写风格（中等互动，亲切感）

### 代表案例
- **金角大王 Alex**（35 赞）：iPhone Notes 截图，AI vibe coding 创业
- **言辞之间**（838 赞）：长文帖，Claude 使用建议

### 视觉特征
- iPhone Notes 应用截图
- 手写字体或系统默认字体
- 黄色便签纸背景
- 非正式、随手记的感觉

### 适用场景
- 快速发帖，不想做设计
- 个人思考、随笔、日记式内容
- 适合测试内容是否有共鸣，再决定是否做精美版

### 技术实现
- 不需要自动化——手动截图即可
- 或者模拟 Notes 风格的 HTML 模板

## 封面选择决策树

```
有视频截帧/出镜素材？
├── 是 → 类型 1（真人 + 大字叠加）——互动天花板最高
└── 否
    ├── 有数字/截图证据？
    │   └── 是 → 类型 3（截图 + 标注）
    └── 否
        ├── 正式内容？
        │   └── 是 → 类型 2（纯文字大字报）——默认选择
        └── 随手记/测试？
            └── 类型 4（Notes 截图）
```

## 对 content-rewriter / videocut cover 的影响

### cover 模板需要支持两种模式

| 模式 | 输入 | 输出 |
|------|------|------|
| **模式 A：真人 + 大字**（默认） | cover.jpg（视频封面/截帧）+ 标题文字 | 1080×1440 图片，人像背景 + 大字叠加 |
| **模式 B：纯文字大字报**（fallback） | 标题文字 | 1080×1440 图片，纯色背景 + 三行大字 |

### 金句卡片（内容页图片）

- 一句金句 ≤ 30 字
- 48-52px 粗体白字
- 深色背景
- 引号装饰
- 每张卡片只有一句话，不是一段文字
